**src/App.tsx — Lift AdminAuthProvider (per D-03, Claude's Discretion)**

Move `<AdminAuthProvider>` from wrapping only the `/admin` route element to wrapping the entire `<Routes>` block inside the `App` function. This gives both `/` and `/admin` routes access to the same auth context instance. The sessionStorage backing means auth state survives the provider re-mounting, but lifting it eliminates the re-mount entirely.

Current structure (lines 48-65):
```
<Routes>
  <Route path="/" element={<AppWithMap apiKey={apiKey} />} />
  <Route path="/admin" element={<AdminAuthProvider>...<AdminAuthProvider>} />
</Routes>
```

Target structure:
```
<AdminAuthProvider>
  <Routes>
    <Route path="/" element={<AppWithMap apiKey={apiKey} />} />
    <Route path="/admin" element={
      <APIProvider apiKey={apiKey} libraries={['places']}>
        <ProtectedRoute><AdminDashboard /></ProtectedRoute>
      </APIProvider>
    } />
    <Route path="*" element={<Navigate to="/" replace />} />
  </Routes>
</AdminAuthProvider>
```
